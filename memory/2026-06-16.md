## Key Decisions

1. API route handlers use NextResponse/NextRequest (not NestJS controllers) — Vercel compatibility
2. Vercel API project: Framework Preset = Next.js (bukan NestJS), build = next build, output = .next/
3. Drizzle client `.query` tidak bisa tanpa schema registry — seed scripts pake `db.select().from()` langsung
4. Supabase client type inference rusak untuk `.select('kolom1, kolom2')` — perlu explicit type casting

## Lesson Learned

- File `vercel.json` override semua settings dashboard — harus dihapus kalo pake framework preset
- Supabase Auth Google OAuth: perlu Client ID + Client Secret, tidak cukup hanya Client ID
- `tsx` tidak auto-load `.env` — perlu explicit env vars passing
- API project perlu `next` sebagai dependency untuk route handler types (NextRequest/NextResponse)
